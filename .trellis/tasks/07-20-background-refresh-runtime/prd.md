# PRD：系统后台任务生命周期

## 目标

为 iOS 15+ 建立可验证的 `BGAppRefreshTask` 注册、提交、续约、取消和完成闭环。

## 需求

1. 使用唯一常量维护任务标识符和最早调度间隔。
2. App 启动时同步注册 handler，Scene 进入后台时按需调度。
3. 无有效登录论坛时不占用后台刷新预算，并取消旧请求。
4. 任务被系统唤醒时先续约，再执行注入的异步同步工作。
5. Expiration 取消工作且任务只完成一次。
6. 注册或提交失败不影响 App 启动和前台功能。

## 验收标准

- [ ] Info.plist 与代码标识符一致。
- [ ] 注册发生在 AppDelegate 启动返回前。
- [ ] Scene 后台入口可调度，任务执行入口可续约。
- [ ] 无账号时取消待执行请求并清理角标。
- [ ] 过期和正常完成不会重复调用任务完成回调。
- [ ] 调度策略具备单元测试。

## 不在范围

- `BGProcessingTask`、后台 URLSession 上传下载和 APNs。
