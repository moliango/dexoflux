# PRD：后台 Topic 更新计数

## 目标

扩展现有 iOS 后台刷新，在不缓存完整 Topic List 的前提下，更新 Home 页“查看 X 个新的或更新的话题”提示。

## 需求

1. 前台成功加载无分类的“最新”第一页后，持久化最多 30 个 Topic 的轻量基线。
2. 后台通知同步成功后，每个已登录论坛额外请求一次“最新”第一页。
3. 使用 Topic ID、回复数、帖子数和最后回复时间识别新增或更新的话题。
4. 待更新 Topic ID 按服务器顺序持久化，最多保存和显示 30 个。
5. Home 页面创建、重新可见或 App 回到前台时恢复后台计算的待更新 ID。
6. 点击现有提示仍使用 `loadIncomingTopics()` 加载，成功后清零并重建基线。
7. 首次没有前台基线时只建立基线，不显示历史更新。
8. Cloudflare、认证、网络、取消或系统过期时保留旧计数，不弹 UI。

## 验收标准

- [ ] 后台每个论坛最多增加一次 Topic List 请求。
- [ ] 无基线时后台刷新结果为 0。
- [ ] 新 Topic、已有 Topic 回复变化能进入待更新列表。
- [ ] 重复后台刷新不会重复计数，数量最多 30。
- [ ] 前后台状态按论坛隔离。
- [ ] Home 回到前台能恢复计数，加载成功后清零。
- [ ] 后台 Topic 请求不触发 WebView 或 Cloudflare 页面。
- [ ] 定向测试和 `build-for-testing` 通过。

## 不在范围

- 持久化完整 Topic、用户、分类或头像数据。
- 后台预取 TopicDetail、图片或更多分页。
- 保证 iOS 后台任务准点执行。
