# PRD：后台通知投递与集成

## 目标

把后台同步快照安全地转换成共享通知游标、跨进程 App 角标和已授权的系统通知，并接入 BGTask 生命周期。

## 依赖

- 依赖 `07-20-background-refresh-runtime` 提供任务生命周期。
- 依赖 `07-20-background-notification-sync` 提供无 UI 同步快照。

## 需求

1. 前后台共用 `baseURL + username` 通知投递游标。
2. 首次快照只建立基线，后续仅投递新增未读通知。
3. 账号角标持久化，冷启动和多论坛情况下能够聚合。
4. 成功同步覆盖账号角标；退出登录或删除论坛清理对应 scope；单论坛失败保留最后已知值。
5. 后台仅在通知权限已授权时投递，不在后台调用授权弹窗。
6. 本地通知继续携带 baseURL、notificationId、topicId 和 postNumber。
7. 将同步引擎接入 BGTask handler，并保持前台协调器行为一致。

## 验收标准

- [ ] 冷后台首轮不会弹历史通知，下一条新通知只弹一次。
- [ ] `.notDetermined` 或 `.denied` 时后台不请求授权且不崩溃。
- [ ] 多论坛角标正确聚合，单论坛失败不清空其他论坛。
- [ ] 删除论坛、退出登录或无有效账号时旧角标可清理。
- [ ] 通知点击路由和前台红点无回归。
- [ ] BGTask 过期取消后不继续投递通知。
- [ ] 集成测试构建与静态检查通过。

## 不在范围

- APNs、后台权限引导页面和用户可见的刷新开关。
