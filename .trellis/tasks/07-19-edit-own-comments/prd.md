# PRD：编辑自己发表的评论

## 目标

允许用户在 TopicDetail 中二次修改服务器允许编辑的自己评论，并在保存后原地更新内容，不改变当前阅读位置。

## 已确认事实

- 现有 `Post` 模型未解码 `raw`、`can_edit`、`yours`。
- 当前只有话题标题编辑，不存在评论正文编辑 API。
- `ReplyComposerViewController` 已具备 Markdown、预览、表情和上传能力，可复用为编辑器。
- `TopicDetailViewModel.reloadPost(postId:)` 可精准刷新单条评论并重新解析正文。
- 帖子是否允许编辑必须以服务端 `can_edit` 为准，不能仅比较用户名。

## 需求

1. 仅当帖子级 `can_edit=true` 时，在帖子更多菜单中显示“编辑”。
2. 点击编辑后获取该帖最新 `raw` Markdown 和实时权限。
3. 复用回复编辑器进入编辑模式，预填原文，标题显示“编辑评论”，主按钮显示“保存”。
4. 保存使用 Discourse `PUT /posts/{id}.json`，参数为 `post[raw]`。
5. 保存成功后 TopicDetail 精准刷新该评论；回复列表弹层重新加载当前回复列表。
6. 保存失败时保留编辑内容和编辑器，不自动关闭。
7. Native Cell 与 WebView fallback 的委托链路保持完整。
8. 文案支持现有国际化语言。

## 验收标准

- [ ] 不可编辑评论的菜单没有编辑入口。
- [ ] 可编辑评论能载入 Markdown 原文并保存二次修改。
- [ ] 更新请求路径、方法和嵌套参数正确。
- [ ] 保存成功后评论原地刷新，TopicDetail 不整页重载、不跳回顶部。
- [ ] Cell 复用后编辑菜单不会残留到无权限评论。
- [ ] API 失败后文本仍保留，可再次提交。
- [ ] 新增模型、路由、编辑模式和权限菜单测试通过。

## 不在范围

- 绕过 Discourse 编辑时限、锁帖或审核权限。
- 编辑他人评论。
- 修改帖子修订历史或填写 edit reason。
