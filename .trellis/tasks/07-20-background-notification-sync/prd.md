# PRD：后台通知同步与提醒

## 目标

在没有可见页面和前台计时器的情况下，同步所有已登录论坛的新通知并更新系统提醒。

## 需求

1. 同步引擎不持有 ViewController，也不临时创建前台通知协调器。
2. 从数据库读取论坛，以持久化 `_t` Cookie 判断登录资格。
3. 官方未读总数和通知列表被转换为纯同步快照，供投递层消费。
4. 后台网络层不展示 Cloudflare 或登录 UI。
7. 支持任务取消、部分论坛失败和空账号结果。
8. 后台 API 请求禁用 WKWebView 会话恢复与响应后的 WebKit Cookie 刷新，认证或 Cloudflare 问题留到前台处理。

## 验收标准

- [ ] 新引擎可独立执行并返回结构化同步结果。
- [ ] 401、403、Cloudflare 或空响应不会在后台创建 WKWebView、清除 Cookie 或弹验证页。
- [ ] 取消后不继续访问后续论坛。
- [ ] 多论坛结果、部分失败和取消策略测试通过。

## 不在范围

- MessageBus 长轮询常驻、APNs、后台 Cloudflare 自动过盾和系统通知 UI。
